# 实验报告：Apple（中国大陆）官方网站前端复刻

---

**实验名称**：Apple 中国大陆官网前端页面复刻  
**实验日期**：2026 年 5 月 31 日  
**实验环境**：Linux (WSL2) | VS Code | HTML5 + CSS3 + JavaScript  
**实验性质**：Web 前端开发综合实验  

---

## 一、实验目的

1. 深入理解大型商业网站（Apple 中国官网）的页面架构、布局策略和设计规范。
2. 掌握从真实网站提取资源（图片、CSS 样式、字体）并本地化重构的技术方法。
3. 实践响应式布局、CSS 动画、视差滚动、毛玻璃导航栏等现代 Web 前端技术。
4. 实现多页面跳转交互，模拟真实网站的用户浏览体验。

## 二、实验内容

本次实验以 [Apple 中国大陆官方网站](https://www.apple.com.cn/) 为参照对象，对其**首页（Homepage）**和 **iPhone 产品子页（iPhone Page）** 进行前端复刻，包括：

1. **页面结构还原**：导航栏、Hero 横幅、产品卡片矩阵、页脚等核心模块。
2. **视觉风格复现**：深色 Hero 区、毛玻璃导航栏、圆角按钮、SF Pro 字体族。
3. **图片资源获取**：从 Apple CDN 服务器下载官方产品图片并本地化引用。
4. **交互动画实现**：视差滚动、元素渐显、悬停效果、平滑锚点跳转。

## 三、实验原理

### 3.1 Apple 官网技术特点

| 技术点 | 说明 |
|--------|------|
| 导航栏 | `position: fixed` + `backdrop-filter: saturate(180%) blur(20px)` 实现毛玻璃效果 |
| Hero 横幅 | 全宽 `692px` 高度，`background-size: cover` 自适应裁剪 |
| 产品网格 | CSS Grid 2 列布局，`12px` 间距 |
| 按钮样式 | `border-radius: 980px`（胶囊形），半透明边框 |
| 动画策略 | CSS `@keyframes` + JavaScript Scroll Observer 触发 |
| 字体 | SF Pro Display / SF Pro Text + PingFang SC 降级方案 |

### 3.2 图片获取原理

通过 cURL 模拟浏览器请求 Apple 中国官网 HTML 和 CSS 文件，正则提取 `background-image: url(...)` 中的图片路径，拼接 `https://www.apple.com.cn` 域名后下载至本地 `images/` 目录。

## 六、CSS 技术要点

### 6.1 CSS 变量体系

```css
:root {
  --btn-blue: #0071e3;        /* 按钮主色 */
  --nav-bg: rgba(0,0,0,0.8);  /* 导航栏半透明 */
  --font-display: "SF Pro Display", "PingFang SC", ...;
  --hero-height: 692px;       /* Hero 区高度 */
  --gap: 12px;                /* 网格间距 */
}
```

### 6.2 关键动画实现

| 动画名称 | 技术方案 | 触发条件 |
|---------|---------|---------|
| Hero 视差 | JS 监听 scroll，动态修改 `backgroundPositionY` | 滚动偏移 × 0.15 |
| Stagger 渐显 | CSS `@keyframes fadeUp` + `.visible` 类名 | Intersection Observer |
| 导航栏加深 | `.scrolled` 类切换 `background` | scrollY > 44px |
| 卡片悬浮 | `transform: scale(1.008)` + `translateY(-4px)` | `:hover` |
| 按钮过渡 | `transition: all 0.25s ease` | 所有交互 |

### 6.3 响应式断点

| 断点 | 适配内容 |
|------|---------|
| > 1068px | 完整桌面布局，3 列网格 |
| 734px – 1068px | 平板：Hero 缩小至 548px，网格变 2 列 |
| < 734px | 手机：单列堆叠，隐藏主导航文字 |

## 八、遇到的问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| Apple 官网无法通过 WebFetch 访问 | 企业安全策略阻止 | 改用 cURL + User-Agent 模拟浏览器 |
| 下载的图片与产品代际不匹配 | 本地截图是 iPhone 17，代码写的是 iPhone 16 | 扫描 images/ 目录后动态适配 |
| emoji 图标与 CDN 图片风格割裂 | 部分卡片用 emoji 占位，部分用真实图片 | 删除 emoji 卡片，保留纯图片卡片 |
| 中文文件名 URL 编码问题 | `17 标准版.png` 含空格和中文 | 使用 `%20` 和 `%E6%A0%87%E5%87%86` 编码 |
| 7 张卡片 3 列布局不对齐 | 7 不能被 3 整除 | 删除 3 张 emoji 卡片后改为 2×2 网格 |

## 九、实验结果

成功复刻了 Apple 中国大陆官网首页和 iPhone 子页的核心布局与视觉风格：

- **首页**包含 3 个全屏 Hero 横幅 + 6 张产品 Promo 卡片 + 完整页脚。
- **iPhone 子页**包含机型选择器、4 张功能卡片、机型比较、配件展示、生态系统、购机优惠等 6 大模块。
- 所有 18 张产品图片均从 Apple CDN 下载，还原度较高。
- 实现了视差滚动、渐显动画、毛玻璃导航、平滑跳转等交互效果。
- 点击首页"进一步了解"按钮可跳转到 iPhone 子页，Nav Logo 可返回首页。

## 十、实验总结

通过本次实验，深入理解了 Apple 官网的设计哲学——**极简、克制、以产品图片为核心**。其技术实现并不复杂，但细节打磨极其到位：0.25s 的按钮过渡、980px 的胶囊圆角、backdrop-filter 毛玻璃、strict sRGB 色彩管理。一个好的商业网站不在于炫技，而在于让用户**感受不到技术的存在**，专注于产品本身。

本次实验也存在不足：Hero 轮播功能未实现（需要 Swiper/Carousel 组件）；缺少对 Apple 智能、隐私等功能详情弹窗的复现；字体因版权问题使用了系统降级方案而非 SF Pro 字体文件。
